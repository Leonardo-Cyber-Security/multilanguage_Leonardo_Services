# Test Account Management

This section defines the process for provisioning test accounts requested by customers.

The process ensures that customer requests for test accounts are handled in a controlled, timely, and secure manner. It includes request intake, validation, approval, provisioning, and delivery.

The customer can start a request via email to the support address listed here: <a href="mailto:lce.support@leonardo.com?subject=Test Account request%20for%20....&body=Insert%20your%20specific%20requests%20following%20the%20instrucions%20">Send an email</a>


## Test Account Request Process

```text
Customer submits request
(via email to the support address)
            │
            ▼
Request Submission
- Purpose of the test account
- Number of accounts required
- Required roles or permissions
- Requester contact details
            │
            ▼
Ticket Registration
- Request logged in the ticketing system
- Ticket assigned for processing
            │
            ▼
Request Review & Validation
- Completeness verification
- Technical feasibility assessment
- Additional information requested if needed
- SLA starts once the request is validated
            │
            ▼
Account Provisioning
- Creation of test account(s)
- Assignment of minimum required permissions
- Application of security policies
- Configuration of expiration settings (if applicable)
            │
            ▼
Verification & Testing
- Functionality checks performed
- Access validation completed
            │
            ▼
Customer Notification
(within 10 business days)
- Credentials or activation instructions provided
- Assigned permissions communicated
- Account validity period specified
- Support contact information shared
            │
            ▼
Request Fulfilled
```

## Limitations

The following are some limitations regarding test account requests:

- Creation of a maximum of 4 small-scale VMs.
- Creation of a maximum of 1 K8S cluster.
- PaaS services can be added on request to the K8S cluster.

Not all PaaS services can be requested in the test environment.
